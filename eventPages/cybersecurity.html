    CYBERSECURITY CHALLENGE - FULL DETAILS

    ADVANCED LEVEL

    The "Phishing Shield"

    Description: An intelligent tool that uses AI to detect sophisticated phishing emails that traditional spam filters miss. It solves the problem of increasingly clever phishing attacks by analyzing email context for unusual activity, not just bad links. The system can also "virtually" visit a suspicious link in a secure sandbox to analyze the landing page for signs of a scam. It then warns the user with a simple, color-coded alert (Green, Yellow, Red) that is easy to understand.

    Key Requirements:

    • AI/ML model for natural language processing (NLP) to analyze email text and context.
    • A secure, isolated sandbox environment to safely render and analyze web pages from suspicious links.
    • Integration with common email clients (e.g., as a browser extension for Gmail or an Outlook add-in).
    • A simple and intuitive user interface for displaying threat levels (e.g., color-coded alerts).
    • Real-time URL scanning and analysis capabilities.

    Outcomes:

    • A significant reduction in successful phishing attacks for users.
    • Increased user awareness and ability to recognize phishing attempts.
    • A measurable accuracy rate (e.g., >95%) in correctly identifying phishing emails while minimizing false positives.

    Conditions:

    • The system must have access to email content for analysis, which requires user consent and strong privacy safeguards.
    • The sandboxing environment must be completely isolated to prevent any potential malware from affecting the user's system.
    • The model must be continuously updated to recognize new and evolving phishing techniques.

    Potential Datasets:

    • PhishTank: A collaborative clearinghouse for data and information about phishing on the Internet. Link to Phishtank
    • Kaggle Phishing Email Dataset: A collection of emails labeled as phishing or legitimate. Link on Kaggle
    • ISCX-URL-2016: A dataset of malicious and benign URLs that can be used for training URL analysis models. Link to UNB Dataset
    Duration: 36 Hours
    Team Size: 1 Member
    Prize Pool: ₹1,000

    INTERMEDIATE LEVEL

    The "Transaction Sentinel"

    Description: A real-time, AI-powered system to combat financial fraud. The model analyzes transaction patterns, user behavior (time of day, location, purchase amount), and device data to instantly flag suspicious activity. The solution would be trained on a dataset of transactions to learn what constitutes normal behavior for a user and would be able to detect anomalies with high accuracy, minimizing false positives and protecting both customers and businesses from financial loss.

    Key Requirements:

    • A real-time data processing pipeline capable of handling high volumes of transactions.
    • An anomaly detection model trained on user-specific and general transaction patterns.
    • Secure integration with financial systems and payment gateways.
    • A dashboard for fraud analysts to review and act on flagged transactions.
    • Compliance with financial data security standards (e.g., PCI DSS).

    Outcomes:

    • A quantifiable reduction in fraudulent transaction losses.
    • A low false-positive rate to ensure legitimate transactions are not blocked unnecessarily.
    • Improved trust and security for customers.

    Conditions:

    • Access to large volumes of anonymized transaction data is essential for training the model.
    • The system must operate with very low latency to approve or flag transactions in real-time.
    • The model must be adaptable to changing fraud patterns.

    Potential Datasets:

    • Kaggle Credit Card Fraud Detection: A highly popular and standardized dataset for fraud detection. Link on Kaggle
    • IEEE-CIS Fraud Detection: A large and comprehensive dataset from a real-world financial services company. Link on Kaggle
    Duration: 36 Hours
    Team Size: 1 Member
    Prize Pool: ₹1,000

    CYBERSECURITY CHALLENGE - FULL DETAILS

    INTERMEDIATE LEVEL

    The "Profile Purity" Detector

    Description: A tool that uses a multi-faceted AI model to identify fake social media accounts. The system would analyze various features of a profile, such as the age of the account, friend-to-follower ratio, the sentiment and coherence of posts (using NLP), and even the profile picture (using a vision model to detect stock photos or AI-generated faces). The tool would then provide a "trust score" for any given social media profile.

    Key Requirements:

    • An NLP model to analyze the text content of posts for sentiment, grammar, and coherence.
    • A computer vision model to analyze profile pictures for signs of being stock photos or AI-generated.
    • A data aggregation module to collect various profile metrics (account age, follower ratio, etc.).
    • A weighted scoring algorithm to combine these features into a single "trust score."

    Outcomes:

    • An accurate and reliable system for identifying fake profiles, bots, and trolls on social media.
    • Improved platform integrity and a safer online environment for users.
    • A clear and understandable trust score to help users make informed decisions.

    Conditions:

    • The tool must operate within the terms of service of the social media platforms it analyzes.
    • The models must be trained on a diverse dataset to avoid bias and ensure accuracy across different types of profiles.
    • Ethical considerations regarding user data privacy must be addressed.

    Potential Datasets:

    • TwiBot-22: A comprehensive dataset for Twitter bot detection. Link to TwiBot-22 on GitHub
    • Kaggle Social Media Fake Profile Detection: A dataset with various features for classifying fake and real accounts. Link on Kaggle
    Duration: 36 Hours
    Team Size: 1 Member
    Prize Pool: ₹1,000
